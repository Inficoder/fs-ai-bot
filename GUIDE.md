## 2. 安装

### 2.1 安装依赖

```bash
cd feishu-ai
npm install
```

### 2.2 编译项目

```bash
# 编译后端
npm run build

# 编译前端管理界面
cd src/web && npx vite build
```

编译成功后，目录下会生成：

- `dist/` — 后端编译产物
- `web-dist/` — 前端管理界面

## 3. 飞书应用创建与配置

### 3.1 创建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/)
2. 点击右上角「开发者后台」
3. 点击「创建应用」→「企业自建应用」
4. 填写应用名称（如"AI 助手"）和描述
5. 创建完成后进入应用管理页面

### 3.2 获取凭证

在应用管理页面左侧菜单：

1. 点击 **「凭证与基础信息」**
2. 记下以下两个值（后续填入 Web 配置页面）：
   - **App ID**（格式：`cli_xxxxxxxxxxxx`）
   - **App Secret**（格式：一串密钥）

### 3.3 添加机器人能力

1. 左侧菜单点击 **「添加能力」→「机器人」**
2. 点击「添加」按钮
3. 配置机器人名称和头像（可选）

### 3.4 配置权限

左侧菜单点击 **「权限管理」**，搜索并开启以下权限：

| 权限 | 说明 |
|------|------|
| `im:message` | 获取群聊消息 |
| `im:message:send_as_bot` | 以机器人身份发送消息 |
| `im:chat` | 获取群组信息 |
| `im:chat:create` | 创建群组（如需 `/create` 命令） |

### 3.5 配置事件（关键步骤）

左侧菜单点击 **「事件与回调」**：

1. 选择 **「长连接」** 模式
2. 添加事件 **「接收消息」**（`im.message.receive_v1`）
3. 点击「确认添加」

> **注意：** 本项目使用 WebSocket 长连接模式，不需要配置公网域名或 HTTPS。长连接模式的好处是不需要暴露公网端口，适合内网部署。

### 3.6 发布应用

1. 左侧菜单点击 **「版本管理与发布」**
2. 点击「创建版本」
3. 填写版本号和更新说明
4. 提交审核
5. 审核通过后，点击「发布」

### 3.7 添加机器人到群

发布后，在飞书客户端：

1. 创建一个群聊或进入已有群
2. 点击群设置 → 「群机器人」→ 「添加机器人」
3. 搜索你的应用名称并添加

或者在 Web 配置页面配置完成后，在群内发送 `/create` 命令自动创建群聊。

## 4. DeepSeek API 申请

1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/)
2. 注册/登录账号
3. 进入 **API Keys** 页面
4. 点击「创建 API Key」
5. 复制生成的 Key（以 `sk-` 开头）

注意保存好 API Key，关闭页面后将无法再次查看完整 Key。

## 5. 启动服务

### 5.1 前台启动

```bash
cd feishu-ai
node dist/cli.js start
```

启动日志示例：

```
服务已启动: http://localhost:8000
[WS] 飞书配置不完整，跳过 WS 连接
```

首次启动会提示配置未完成，这是正常的——需要先通过 Web 页面填写飞书和 AI 参数。

### 5.2 配置完成后重启

在 Web 页面填写配置后，需要重启服务使配置生效：

```bash
# 停止
node dist/cli.js stop

# 启动
node dist/cli.js start
```

启动成功日志：

```
服务已启动: http://localhost:8000
[WS] 飞书长连接已建立
```

出现「飞书长连接已建立」表示与飞书的 WebSocket 连接成功，可以开始在群内使用机器人。

### 5.3 查看状态

```bash
node dist/cli.js status
```

输出 `运行中 (PID: xxxx)` 或 `已停止`。

## 6. Web 配置页面

服务启动后，浏览器访问：

```
http://localhost:8000
```

### 6.1 配置项说明

| 配置项 | 必填 | 说明 |
|--------|------|------|
| **App ID** | 是 | 飞书开放平台 → 凭证与基础信息 |
| **App Secret** | 是 | 飞书开放平台 → 凭证与基础信息 |
| **Verification Token** | 否 | Webhook 模式的校验 token，长连接模式可留空 |
| **API 地址** | 否 | 飞书 API 地址，默认 `https://open.feishu.cn/open-apis`（国际版使用 `open.larksuite.com`） |
| **API Key** | 是 | DeepSeek 平台申请的 API Key |
| **API 地址** | 否 | DeepSeek API 地址，默认 `https://api.deepseek.com` |
| **模型** | 否 | 默认 `deepseek-chat`，可选 `deepseek-reasoner`、`deepseek-v4-pro`、`deepseek-v4-flash` |
| **刷新间隔** | 否 | 流式回复卡片更新间隔(毫秒)，默认 500 |
| **累积 Token** | 否 | 每次刷新累积的 token 数，默认 20 |
| **API 超时** | 否 | DeepSeek HTTP 超时(秒)，默认 120 |
| **对话超时** | 否 | 单次对话任务超时(秒)，默认 120 |
| **HTTP 端口** | 否 | 服务端口，默认 8000 |

## 7. 飞书群使用

配置完成后，在飞书群内 @机器人 发送消息即可使用。

### 7.1 可用命令

| 命令 | 说明 |
|------|------|
| `@机器人 你的问题` | AI 对话，以流式卡片回复 |
| `/help` | 显示帮助信息 |
| `/system <提示词>` | 设置 AI 角色（如 `你是一个 Python 专家`） |
| `/model [名称]` | 查看或切换 AI 模型 |
| `/temp <0.0-2.0>` | 调整创意度（0=精确，1=平衡，2=创意） |
| `/search [on\|off]` | 开启/关闭智能搜索 |
| `/create <名称>` | 创建新群聊并自动建立对话上下文 |
| `/reset` | 清空当前对话历史 |
| `/list` | 列出所有活跃对话 |
| `/delete` | 删除当前对话上下文 |
| `/note <内容>` | 添加备注 |

### 7.2 流式回复

AI 回复以**消息卡片**形式逐段展示：

1. 发送消息后，立即出现「思考中...」卡片
2. AI 开始生成回复，卡片内容逐步更新
3. 生成完成后显示完整回复

### 7.3 智能搜索

开启智能搜索后（`/search on`），AI 会自动联网检索相关信息，再结合搜索结果回答。适用于需要实时信息的场景。

## 常见问题

**Q: 启动后「飞书长连接已建立」没有出现？**
A: 检查 Web 配置页面的 App ID 和 App Secret 是否正确填写，保存后重启服务。

**Q: 群内 @机器人 没有响应？**
A: 确认应用已发布、已添加到群、以及飞书开放平台的事件配置已保存（长连接模式）。

**Q: 重启服务后收不到消息？**
A: 如果使用长连接模式，重启后通常会自动重连。如仍收不到，可以去飞书开放平台「事件与回调」页面重新保存一次配置。

**Q: 如何修改端口？**
A: 方式一：在 Web 配置页面修改端口并保存，重启服务。方式二：启动时指定 `feishu-ai start -p 8080`。
